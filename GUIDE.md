# 弹幕排队姬 使用教学

> 本文面向主播和管理员，介绍如何配置和使用弹幕排队姬。

---

## 第 1 章：快速开始

### 系统要求

- Windows 10 / 11（64 位）
- Python 3.10+（源码运行）或直接使用打包的 `main.exe`

### 安装依赖（源码运行）

```bash
pip install Pillow qrcode brotli psutil pyyaml protobuf
```

### 启动方式

**方式 A：使用打包版 exe**

下载 Release 中的 zip，解压后运行 `main.exe`。

**方式 B：源码启动**

```bash
cd bilipdj          # 进入源码包目录
python core/control_panel.py
```

启动后程序会自动打开控制台窗口，后端服务默认运行在 `http://127.0.0.1:9816`。

## 第 3 章：B 站直播接入

### 步骤 1：获取 Cookie（扫码登录）

1. 点击顶栏「打开配置页」，在浏览器打开配置页面。
2. 找到「Bilibili 登录」区域，点击「扫码登录」。
3. 使用 Bilibili App 扫描二维码并确认登录。
4. 登录成功后 Cookie 和 UID 会自动回填到配置页。

> Cookie 有效期通常为数月。过期后重新扫码即可。

### 步骤 2：填写直播间信息

在控制台「设置」标签页 → 「平台参数」→「B站参数」：

| 字段 | 说明 |
|---|---|
| 直播间号 | 直播间 URL 中的数字，如 `https://live.bilibili.com/12345` 中的 `12345` |
| UID | 你的 B 站用户 ID（扫码后自动填入） |
| Cookie | 登录态（扫码后自动填入） |

### 步骤 3：保存并启动

1. 在「设置」页确认平台选择为「bilibili」。
2. 点击「保存配置」。
3. 点击顶栏「启动后端」，等待日志出现 `Bilibili connected room_id=...`。

## 第 4 章：抖音直播接入

### 步骤 1：启用抖音配置

在「设置」→「平台参数」：
1. 将「当前平台」切换为「douyin」。
2. 勾选「启用抖音配置」。

### 步骤 2：一键获取参数

在「抖音参数」区域顶部：

1. 在「直播间链接」输入框粘贴抖音直播间 URL，例如：
   ```
   https://live.douyin.com/833045918808?room_id=7628150693462838016
   ```
2. 如果已有 Cookie 请先填入「Cookie」字段（可选，有助于提高解析成功率）。
3. 点击「获取参数」，等待状态栏显示主播昵称和直播状态。
4. `live_id`、`room_id`、`user_id` 等字段会自动回填。

> **live_id vs room_id**：URL 路径中的数字（如 `833045918808`）是分享 ID（live_id），
> URL 中 `?room_id=` 后面的数字才是 WebSocket 房间 ID（room_id）。
> 两者都需要正确填写，一键获取会自动处理。

### 步骤 3：获取抖音 Cookie（可选但推荐）

Cookie 可以提升参数获取成功率并支持更完整的弹幕功能：

1. 在浏览器（Chrome/Edge）打开抖音直播间页面并登录。
2. 按 F12 打开开发者工具 → Network（网络）面板。
3. 刷新页面，找到任意 `live.douyin.com` 的请求。
4. 在请求头（Request Headers）中找到 `Cookie`，复制全部内容。
5. 粘贴到「抖音参数」→「Cookie」字段。

### 步骤 4：保存并启动

1. 点击「保存配置」。
2. 点击「启动后端」，等待日志出现 `Douyin connected live_id=...`。

## 第 5 章：排队指令

观众在直播间发送以下弹幕即可触发排队功能。

### 普通观众

| 发送内容 | 效果 |
|---|---|
| `排队` | 以昵称加入排队 |
| `官服排` / `官服排队` | 以 `官\|昵称` 加入（官方服务器标识） |
| `B服排` / `排B服` | 以 `B\|昵称` 加入（B 服标识） |
| `超级排` / `超级排队` | 以 `<昵称>` 加入（超级/特殊标识） |
| `小米排` / `排米服` | 以 `米\|昵称` 加入（米服标识） |
| `排队 自定义内容` | 以自定义内容加入排队 |
| `取消排队` | 退出排队 |
| `替换 新内容` | 修改自己的排队内容 |
| `修改 新内容` | 同上 |

### 管理员 / 主播指令

| 发送内容 | 效果 |
|---|---|
| `完成` | 删除队列第一人 |
| `del 昵称` | 删除指定用户 |
| `add 昵称 内容` | 在队首手动插入一条 |
| `无影插 昵称` | 静默插队（不广播通知） |
| `暂停排队功能` | 关闭排队总开关 |
| `恢复排队功能` | 开启排队总开关 |
| `设置排队上限 N` | 设置最大人数（N 为整数） |

### super_admin 专属

| 发送内容 | 效果 |
|---|---|
| `添加管理员 昵称` | 将该用户提升为管理员 |
| `取消管理员 昵称` | 将该用户降为普通成员 |
| `拉黑 昵称` | 将该用户加入黑名单 |
| `取消拉黑 昵称` | 将该用户移出黑名单 |

## 第 6 章：队列管理

### GUI 操作（当前排队 Tab）

在列表中选中条目后：

| 按钮 | 功能 |
|---|---|
| 删除 | 移除选中条目 |
| 上移 | 将选中条目向前移一位 |
| 下移 | 将选中条目向后移一位 |
| 在下方新增 | 在选中位置下方手动插入一条 |
| 一键清空 | 清空整个队列（双击确认） |
| 刷新 | 手动刷新队列列表 |

### 存档槽位

存档槽位用于保存多个独立的队列状态，例如按游戏房间、按时段分开存档。

- 共 10 个槽位，在「设置」→「基础设置」→「当前排队存档」中切换。
- 切换后自动恢复该槽位上次保存的队列。
- 每次队列变化（排队/取消/删除）都会自动保存到当前槽位。

## 第 7 章：透明弹窗（OBS）

透明弹窗可以将队列悬浮显示在游戏画面上，OBS 通过窗口捕获识别并透明处理。

### 启动弹窗

在「透明窗口」标签页点击「启动弹窗」。

### OBS 配置

1. 在 OBS 的「来源」面板点击 `+` → 「窗口捕获」。
2. 窗口选择标题为 **排队透明弹窗** 的窗口。
3. 勾选「允许透明」（Allow Transparency）。
4. 拖动来源调整在画面中的位置。

### 尺寸与缩放

在「透明窗口」标签页：

| 设置项 | 说明 |
|---|---|
| 宽度 / 高度 | 弹窗像素尺寸（建议与 OBS 来源大小一致） |
| 文字缩放(%) | 队列字体大小，40%~250%，默认 50% |

调整后点击「应用到透明窗」立即生效。

也可以直接拖拽弹窗边框进行缩放。

### 样式自定义

在「样式设置」标签页可以调整：
- 文字颜色、描边颜色、是否开启描边
- 字体大小、粗体/斜体
- 背景颜色

## 第 8 章：权限与开关

### 权限层级

| 层级 | 说明 |
|---|---|
| `super_admin` | 最高权限，可管理管理员名单，只建议主播账号使用 |
| `admin` | 可执行所有队列操作（删除、插队等） |
| `jianzhang` | 舰长，开启对应开关后可插队 |
| `member` | 普通会员，只能操作自己的排队 |
| `blacklist` | 黑名单，所有指令被屏蔽 |

在「权限」标签页可以直接编辑各角色名单，每行一个用户昵称，点「保存」后立即生效。

### 功能开关

在「开关」标签页可以开启/关闭以下功能：

| 开关 | 默认 | 说明 |
|---|---|---|
| 排队总开关 | ✅ | 关闭后所有排队指令均不响应 |
| 官服排队 | ✅ | 允许 `官服排` 指令 |
| B 服排队 | ✅ | 允许 `B服排` 指令 |
| 超级排队 | ✅ | 允许 `超级排` 指令 |
| 米服排队 | ✅ | 允许 `小米排` 指令 |
| 取消排队 | ✅ | 允许 `取消排队` 指令 |
| 修改排队内容 | ✅ | 允许 `替换/修改` 指令 |
| 舰长插队 | ❌ | 开启后舰长可以插队到队列前部 |
| 房管拥有管理员权限 | ❌ | 开启后 B 站房管等同于 admin |

## 第 9 章：常见问题

### 弹幕连接失败

**B 站：**

1. 确认直播间号填写正确（是数字，不是主播名）。
2. 确认 Cookie 有效。Cookie 过期时日志会出现 `auth failed` 或 `token invalid`，需重新扫码。
3. 检查网络是否可以访问 B 站。

**抖音：**

1. 确认 live_id 和 room_id 都已正确填写（两者不同，见第 4 章）。
2. 尝试使用「一键获取参数」重新抓取。
3. 填写有效的 Cookie 可以提高成功率。

### 抖音"没有弹幕"

按以下顺序自查：

1. 检查「room_id」是否填写了 URL 中 `?room_id=` 后的数字（不是路径中的数字）。
2. 确认直播间正在直播（工具只接收直播中的弹幕，直播已结束会自动等待）。
3. 确认「启用抖音配置」已勾选。
4. 尝试重新获取参数，或重启后端。

### Cookie 过期

- **B 站**：顶栏点「打开配置页」→ 重新扫码登录。
- **抖音**：按第 4 章步骤 3 重新从浏览器复制 Cookie。

### 端口冲突

如果 9816 端口被占用，在「设置」→「基础设置」→「监听端口」中修改为其他端口，保存后重启后端。

浏览器访问地址也需要对应修改（将 `9816` 替换为新端口）。

### 日志出现乱码

确认终端或 GUI 使用 UTF-8 编码。Windows 系统可以在「设置」→「日志等级」调为 `DEBUG` 查看详细信息。
