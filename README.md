# Bilipdj — GUI 控制面板使用说明

Bilipdj 是一个给 Bilibili 直播间用的弹幕排队工具。观众在直播间发送排队相关弹幕，后端会自动维护队列，GUI 控制面板负责启动服务、修改配置、查看日志、调整权限和功能开关。

这份 README 只讲一件事：GUI 控制面板怎么用。

## 先做什么

启动控制面板：

```bash
python bilipdj/core/control_panel.py
```

打开后，最上面有 4 个按钮：

- `启动后端`：启动排队后端服务。
- `停止后端`：停止后端服务。
- `配置页`：打开浏览器里的 Web 配置页面。
- `排队展示页`：打开浏览器里的排队展示页面。

右侧状态文字会显示当前后端是否已启动。

建议第一次使用按这个顺序操作：

1. 先到 `设置` 标签页填写直播间信息。
2. 点击 `保存配置`。
3. 点击 `启动后端`。
4. 回到 `日志` 标签页确认是否连上直播间。
5. 点击 `排队展示页`，把页面放到浏览器或 OBS 里显示。

## 标签页怎么用

### 1. 日志

这个标签页主要用来看后端有没有正常工作。

你会看到：

- 顶部直播间连接状态。
- 下方实时滚动日志。

常见状态含义：

- `后端未启动`：还没点启动，或者进程已经退出。
- `等待弹幕流`：后端启动了，但还没成功接上直播间弹幕。
- `已连接`：已经连上直播间，后面会显示当前 WebSocket 客户端数量。

第一次排查问题时，先看这个页面。

### 2. 当前排队

这个标签页用来查看当前队列。

你会看到：

- 当前排队人数。
- 队列列表。
- `刷新` 按钮。

后端运行时会自动刷新。这里适合确认：

- 用户有没有成功进队。
- 当前顺序对不对。
- 切换存档槽位后，恢复出来的是哪一组队列。

### 3. 设置

这是最重要的标签页，第一次使用基本都从这里开始。

可编辑内容包括：

- `监听地址`
- `监听端口`
- `直播间号`
- `UID`
- `Cookie`
- `日志保留天数`
- `日志等级`
- `日志存档槽位`
- `启用排队存档`
- `启动时自动运行后端`

各项怎么理解：

- `监听地址`：一般保持 `0.0.0.0` 即可。
- `监听端口`：默认 `9816`，如果端口冲突再改。
- `直播间号`：你的 Bilibili 直播间房间号。
- `UID`：主播 UID。
- `Cookie`：Bilibili 登录 Cookie，后端连接和接口调用要用。
- `日志保留天数`：过期日志会自动清理。
- `日志等级`：平时用 `INFO`，排查问题时可改成 `DEBUG`。
- `日志存档槽位`：选择当前要使用的队列存档槽位，范围 1 到 5。
- `启用排队存档`：打开后，队列会写入 CSV 存档。
- `启动时自动运行后端`：勾选后，GUI 打开时会自动拉起后端。

下面两个按钮：

- `保存配置`：把当前内容写入配置文件，并同步当前存档槽位。
- `刷新配置`：重新从本地配置文件读取。

第一次最少要填这 3 项：

- `直播间号`
- `UID`
- `Cookie`

如果这 3 项没填对，后端通常无法正常接入直播间。

### 4. 权限

这个标签页用来配置谁有管理权限。

当前有 4 个文本框：

- `super_admin`：最高管理员，可新增和取消管理员，权限最高。
- `admin`：管理员，拥有大多数管理权限。
- `jianzhang`：舰长名单，仅用于舰长权限相关逻辑。
- `member`：普通成员。

使用方法：

- 每行输入一个用户名。
- 改完后点击 `保存权限`。
- 想放弃当前修改，就点 `刷新权限` 重新读取。

建议至少先填一个 `super_admin`，否则后面很多管理命令没人能执行。

如果后端正在运行，保存后会立即生效；如果后端没运行，就会先写到本地文件，等下次启动时生效。

### 5. 开关

这个标签页用来控制哪些功能允许使用。

当前可以控制的开关包括：

- 普通排队
- 官服排队
- B服排队
- 超级排队
- 米服排队
- 取消排队
- 修改排队内容
- 舰长插队
- 允许房管执行管理命令

使用方法很直接：

- 勾选表示启用。
- 取消勾选表示禁用。
- 改完点 `保存开关`。

这个页面适合做直播前的快速限制。例如：

- 不想让舰长插队，就把 `舰长插队` 关掉。
- 不想让房管代替管理员操作，就把 `允许房管执行管理命令` 关掉。

### 6. 样式设置

这个标签页用来改排队展示页的外观。

可以修改：

- 背景渐变色
- 页面文字颜色
- 队列字体大小
- 文字渐变起始色和结束色
- 文字描边颜色

右侧会有一个近似预览。

按钮说明：

- `保存样式`：保存当前展示样式。
- `恢复默认`：恢复到默认外观。

如果你想把展示页做成更适合 OBS 的视觉效果，就用这个标签页调。

### 7. 关于

这个标签页主要显示版本和软件说明，不涉及配置。

## 一个最常见的使用流程

如果你只是想尽快用起来，按下面做：

1. 打开 GUI。
2. 进入 `设置`，填好 `直播间号`、`UID`、`Cookie`。
3. 点击 `保存配置`。
4. 进入 `权限`，给自己加一个 `super_admin`。
5. 按需在 `开关` 里决定是否允许舰长插队、房管管理。
6. 点击 `启动后端`。
7. 回到 `日志`，确认状态变成 `已连接`。
8. 点击顶部 `排队展示页`，在浏览器中查看展示效果。
9. 如需调整展示外观，进入 `样式设置` 修改。

## 保存后什么时候生效

- `设置`：保存后写入配置文件；部分运行参数需要后端继续运行或重新连接后体现。
- `权限`：后端运行中保存，立即生效。
- `开关`：后端运行中保存，立即生效。
- `样式设置`：保存后展示页会按新样式读取。

## 常见问题

### 后端启动了，但一直是“等待弹幕流”

优先检查：

- `直播间号` 是否正确。
- `UID` 是否正确。
- `Cookie` 是否有效。

再看 `日志` 标签页里的报错内容。

### 保存权限或开关时报错

通常是后端没运行或暂时无响应。这个 GUI 会尽量回退到本地文件保存，所以你保存后下次启动后端时仍会生效。

### 排队展示页没有内容

先确认：

- 后端是否已启动。
- `日志` 里是否显示已连接。
- `当前排队` 标签页里是否已经能看到队列。

如果 `当前排队` 里都没有内容，说明问题不在展示页，而在后端接收弹幕或排队逻辑。
